Automated Reconnaissance Pipeline

Recon Automation Tool

Automated reconnaissance pipeline engineered for cybersecurity professionals and bug bounty hunters. Multi-stage enumeration to vulnerability reporting — single command execution.

Critical High Medium Low Info
// scan pipeline
Subdomain Enum → Dedup → httpx Probe → Live Filter 200 → Nuclei Scan → Report

01 Features

Enumeration

Multi-tool subdomain discovery via subfinder, assetfinder, findomain, chaos, and crt.sh. Automatic deduplication across all sources.

Vulnerability Scanning

Nuclei with automatic template selection (-as), all severities, configurable rate limiting, and dual TXT + JSONL output.

Live Host Probing

httpx probing to identify live hosts. Full severity breakdown displayed after each scan completion.

Flexible Input

CLI args, text files, JSON files, interactive mode. Bugcrowd target scraping integration included out of the box.

Structured Reporting

Formatted scan summary reports. JSONL output for downstream SIEM or pipeline integration.

VPS / CI Ready

Non-interactive mode, log file output, graceful signal handling. Cross-platform: Windows, macOS, Linux.

02 Prerequisites

Python 3.8+ Go 1.21+ httpx & nuclei — mandatory

Go 1.21+ is required for compiling and installing the core recon toolchain. Ensure ~/go/bin is on your $PATH before proceeding.

03 Installation

Step 1 — Clone the Repository

$ git clone <repository-url>
$ cd "Recon automation"

Step 2 — Install Python Dependencies

$ pip install -r requirements.txt

Step 3 — Install Recon Tools

httpx and nuclei are mandatory. subfinder, assetfinder, findomain, and chaos are optional but significantly improve coverage.

Windows
# 1. Install Go from https://go.dev/dl/ (.msi installer)
# 2. Open a new terminal and run:

$ go install -v github.com/projectdiscovery/httpx/cmd/httpx@latest
$ go install -v github.com/projectdiscovery/nuclei/v3/cmd/nuclei@latest
$ go install -v github.com/projectdiscovery/subfinder/v2/cmd/subfinder@latest
$ go install -v github.com/tomnomnom/assetfinder@latest
$ go install -v github.com/projectdiscovery/chaos-client/cmd/chaos@latest

Install findomain: download the Windows binary from github.com/Findomain/Findomain/releases and add it to PATH.

!
Path Configuration — Windows

If any tool shows 'command' is not recognized, add %USERPROFILE%\go\bin to your system PATH. Run as Administrator:

# Check Go bin path
$ go env GOPATH

# Add to PATH (PowerShell, run as Administrator)
$ [Environment]::SetEnvironmentVariable("Path", $env:Path + ";" + (go env GOPATH) + "\bin", "User")

04 Usage

Interactive Mode — Windows / macOS

$ python recon.py

Launches a menu to scrape Bugcrowd, enter domains manually, or load from a file.

CLI Mode — All Platforms

# Single domain
$ python recon.py -d example.com

# Multiple domains (comma-separated)
$ python recon.py -d example.com,target.org,test.io

# Load from text file (one per line)
$ python recon.py -dL domains.txt

# Load from JSON file
$ python recon.py --json-file targets.json

# Scrape Bugcrowd targets
$ python recon.py --bugcrowd

# Custom output directory
$ python recon.py -d example.com --output-dir /path/to/output

VPS Usage

Always use CLI arguments on a VPS (no interactive mode without TTY). Use --log-file to persist output across SSH disconnects.

Background with nohup

$ nohup python recon.py -d example.com --log-file recon.log &

Inside tmux or screen

$ tmux new -s recon
$ python recon.py -dL targets.txt --log-file scan.log
# Ctrl+B, D to detach  |  tmux attach -t recon to reconnect

Cron (scheduled scan)

$ crontab -e

# Daily scan at 02:00
0 2 * * * cd /opt/recon-automation && python recon.py -dL targets.txt --log-file /var/log/recon.log

07 JSON Input Formats

The --json-file flag accepts multiple formats:

// flat list
["example.com", "target.org"]
// object list
[
  {"domain": "example.com"},
  {"domain": "target.org"}
]
// bugcrowd-style
{
  "program_name": [
    {"domain": "*.example.com"},
    {"domain": "target.org"}
  ]
}

10 Troubleshooting

✕
"MISSING REQUIRED TOOLS: httpx, nuclei"

Go tools are not in your PATH. Run echo $PATH and ensure it includes ~/go/bin or $(go env GOPATH)/bin.

!
httpx or nuclei not found after installation

Close and reopen your terminal, or run source ~/.bashrc (Linux) / source ~/.zshrc (macOS).

i
crt.sh errors (502, timeout)

The crt.sh API is occasionally slow or down. This is handled gracefully and does not affect other tools.

i
No live subdomains found

httpx filters for 200 OK responses only. The target may return 301, 403, etc. Check httpx_output.txt for all responses.

✕
Permission denied on VPS

Ensure Go binaries have execute permissions: chmod +x ~/go/bin/*
